Key parameters in notebook Cell 4:

CONFIG = {
    'device': 'cuda' or 'cpu',
    'max_length': 512,
    'batch_size': 16,
    'embedding_dim': 384,
    'lstm_hidden_dim': 256,
    'lstm_num_layers': 2,
    'num_epochs': 5,
    'learning_rate': 0.001,
    'top_k_retrieval': 5,
    'num_eval_questions': 100,
}

ADJUSTMENTS:
- Low GPU memory: batch_size=8, num_eval_questions=50
- Fast test: num_epochs=3, num_eval_questions=20
- High quality: num_epochs=10, top_k_retrieval=10

================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

LSTM ARCHITECTURE
-----------------
Input (token_ids) → Embedding (384)
                  ↓
Bidirectional LSTM (256×2 hidden, 2 layers)
                  ↓
Mean Pooling
                  ↓
FC Layer → Output (embedding_dim)
                  ↓
L2 Normalization

RAG PIPELINE
------------
Question → Encode (SentenceTransformer)
         ↓
ChromaDB Search (Top-K)
         ↓
Context Assembly
         ↓
LLM Generation (ChatGPT/Llama)
         ↓
Answer + Metrics

EVALUATION FLOW
---------------
For each question:
    1. Retrieve documents (measure time)
    2. Generate answer (measure time)
    3. Compute ROUGE vs reference
    4. Detect hallucination (token overlap)
    5. Record all metrics

Aggregate:
    - Mean ± Std for all metrics
    - Per-system comparison
    - Statistical analysis
    - Visualizations
